## 1. What this project is

Headless WordPress + Next.js 16 marketing site for **AICONIQ**.

- **Production**: `https://aiconiq.io/` (WP backend + Next.js front-end on Michael's server, behind nginx).
- **Frontend (homepage + articles + imprint)**: rendered by Next.js. Source lives in `wp-content/themes/e-theme/assets/`.
- **Frontend (subpages with the "Blocks (PHP-rendered)" template)**: rendered server-side by PHP from `wp-content/themes/e-theme/page-templates/blocks.php` and the per-block partials in `page-templates/blocks/`. **This is the PHP port we added in May 2026.**
- **Backend**: WordPress with the custom `e-theme`. ACF Pro Flexible Content for `body_blocks`. Polylang Pro for EN/DE.

Two rendering paths exist intentionally:
1. **Next.js path** — for the marketing home and React-heavy content.
2. **PHP path** — for sub-pages where the client wants pure WP rendering (no Next.js involved). Same ACF blocks, same Tailwind look, just rendered by PHP + a tiny vanilla JS bundle.

## 4. How to set up the project on a fresh machine (Andrey, this is for you)

1. **Clone**:
   ```bash
   git clone https://git.travely24.com/ai/aiconiq.git C:/OSPanel/home/aiconiq-repo
   cd C:/OSPanel/home/aiconiq-repo
   git checkout wppage-nextjs
   ```

2. **Install OSPanel** (or any local WP stack — XAMPP, MAMP, Local…). Point a virtual host to `C:/OSPanel/home/aiconiq-repo/` (WP root).

3. **Create a fresh WP database**, install WordPress as usual. Then activate the theme **e-theme** (`wp-content/themes/e-theme`).

4. **Install required WP plugins** (must be the **Pro** versions):
   - ACF Pro
   - Polylang Pro (configure 2 languages: `en` and `de`)

5. **Run the seeder** to populate content (one click):
   - Open WP admin → **Tools → "AICONIQ: Seed home page"** → click **"Run seeder"**.
   - This downloads images from the bundled `assets/public/` folder (and from `aiconiq.io` for heavy videos), creates EN + DE home pages with all 17 body_blocks, creates EN + DE **Test Page** pages with the same 17 blocks, creates the Primary + Footer menus, and writes ACF footer settings.
   - The seeder is idempotent — re-running just refreshes content.

6. **Build the theme CSS** for the PHP-rendered side (Tailwind):
   ```bash
   cd wp-content/themes/e-theme/assets
   npm install
   npm run build:theme-css
   ```
   This compiles `src/theme.css` → `dist-php/theme.css`. The file is git-tracked so it ships pre-built, but rebuild after editing styles or block templates.

7. **(Optional) Run Next.js dev server** for the React side:
   ```bash
   cd wp-content/themes/e-theme/assets
   npm run dev
   # → http://localhost:3000
   ```

8. **Verify in the browser**:
   - WP homepage: `http://<your-host>/en/` (React via Next, OR via PHP if you disable `front-page.php`)
   - WP test page (PHP-rendered, what we built): `http://<your-host>/en/test-page/` — should show all 17 blocks.
   - DE: `http://<your-host>/de/test-page/`

> Yes, the answer to "can I just clone and continue?" is **yes** — clone, install plugins, run seeder. Database content is regenerated by the seeder, so no DB dump is needed.

## 7. The PHP renderer in 30 seconds

- A WP page using the **"Blocks (PHP-rendered)"** template (`page-templates/blocks.php`) is dispatched by `aiconiq_render_block()` (`functions/php-render-helpers.php`).
- For each block in `body_blocks`, `blocks.php` includes the matching partial in `page-templates/blocks/{type}.php` and exposes `$data`, `$anchor`.
- Each partial uses Tailwind classes (compiled via `npm run build:theme-css`) for styling, plus a few `data-aiconiq-*` attributes that `dist-php/theme.js` hooks into for interactivity (video autoplay, slider, mobile menu, sticky header, count-up, Security sticky image, Calendly popup).
- Hover effects are pure CSS in `src/theme.css`.
- Knowledge step cards have CSS-only hover animations mirroring the React framer-motion versions: card1 (chart rows slide left), card2 ("empty cards" fly in from edges + trigger row borders glow), card3 (avatar strip slides), card4 (image crossfade + review graph rises).
- ConsultantSection: at rest avatars are parallel, on hover they fan out (7°/2°/-11°). The back_logo "star" never moves.

## 8. Commands cheat-sheet

```bash
# Build the PHP-side CSS (after editing src/theme.css or any *.php block)
cd wp-content/themes/e-theme/assets
npm run build:theme-css

# Watch mode while iterating
npm run watch:theme-css

# Next.js dev (the React side)
npm run dev                # → http://localhost:3000

# Next.js production build
npm run build

# Run seeder (or open Tools → "AICONIQ: Seed home page" in WP admin)
# The seeder is idempotent.
```

If you delete a Next.js route or edit `tsconfig.json`, you may need:

```bash
rm -rf wp-content/themes/e-theme/assets/.next
```

before re-building, otherwise stale `.next/dev/types/validator.ts` may reference deleted files.
