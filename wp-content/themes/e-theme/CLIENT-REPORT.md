# Отчёт по интеграции AICONIQ + WordPress

Front-end проект (Next.js 16) интегрирован с WordPress: контент главной страницы теперь полностью управляется через админку без правки кода. Использован headless-подход — WordPress хранит данные, Next.js остаётся боевым сайтом для пользователей.

## Что теперь умеет редактор в админке

**Главная страница**

- 17 типов блоков (Hero, Automation vs Collaboration, Positioning, Problem & Solution, Knowledge intro/steps, Lead capture top/bottom, Solutions, Stats, Testimonials, Security, Built by humans, Competitive advantage, Consultant, плюс универсальные Free text и Divider).
- Каждый блок добавляется/удаляется/перетаскивается в админке. Поля под каждый блок — текст, картинки, видео, кнопки, повторяющиеся элементы (карточки, шаги, отзывы).
- Кнопки во всех релевантных блоках имеют переключатель **«Open chat / External URL»** — одним кликом превращаются в ссылку без правки кода.
- Поля Stats теперь поддерживают до 4 элементов сверху и до 4 снизу (раньше нижний блок был ограничен 3).

**Двуязычие**

- Контент полностью двуязычный (EN/DE), связан через Polylang Pro.
- Кнопка **«Sync block order to translations»** в правом сайдбаре редактора страницы — синхронизирует порядок блоков из текущего языка во все переводы одним кликом. Тексты и медиа в каждом блоке остаются независимыми, двигается только порядок.

**Меню**

- Меню хедера и меню футера управляются стандартно через **Appearance → Menus** (Primary EN/DE, Footer EN/DE).
- Любой пункт может быть якорной ссылкой (`#solutions`) — фронтенд плавно скроллит к нужному блоку. Внешние ссылки тоже поддерживаются.

**Футер**

- Все настройки футера (логотипы, описание, email, соцсети, ссылки) — на отдельной странице **Footer** в верхнем меню админки. Раздельные страницы EN/DE.

**Заполнение контента с нуля**

- Кнопка **Tools → AICONIQ: Seed home page → Run seeder** одним нажатием создаёт обе языковые версии главной, заливает медиа, настраивает меню и футер из эталонных текстов с aiconiq.io. Удобно для развёртывания на новом сервере.

## Архитектурные улучшения

- ACF-поля блоков ограничены **только** страницами с шаблоном «Blocks (Next.js)». На статьях, обычных страницах и блоге UI редактора не меняется.
- Сидер сначала ищет медиа в локальной папке проекта (большая часть ассетов закоммичена в репо) и только потом обращается к aiconiq.io. Команда `npm run fetch-media` дополнительно скачивает недостающие тяжёлые видео — после этого первичное развёртывание возможно вообще без интернета.
- Footer-навигация теперь читается из WP-меню (раньше была захардкожена в коде) — один источник правды для шапки и футера.

## Что доступно для следующей итерации

- В Next.js 16 появилось предупреждение про `middleware.ts` (нужно переименовать в `proxy.ts`) и про невалидный сегмент в одном роуте — не блокируют сборку, но стоит почистить отдельной задачей.
- Один из видео-форматов в Hero и в Auto-vs-intro можно сжать, сейчас тянутся MP4 на 30+ МБ — заметно влияет на первый paint на медленных каналах.
- Если бизнес-команда захочет поменять Calendly URL без перезапуска сборки — вынести его в Footer Options или в отдельный «Site Settings» Options Page.

## Где лежит код

Ветка `wppage-nextjs` в `git.travely24.com/ai/aiconiq`. Развёрнутая инструкция для разработчика — `wp-content/themes/e-theme/SETUP.md`. Состояние и обоснование архитектурных решений — `wp-content/themes/e-theme/HANDOVER.md`.
